# 📊 데이터 유형 분류: 정형 · 반정형 · 비정형

> 싸이월드, 페이스북, IoT, 주식 등 실제 사례 중심의 완전 분류 가이드

---

## 🗂️ 전체 분류 개요

```
빅데이터
├── 정형 데이터 (Structured Data)
│   ├── 관계형 DB (RDBMS) 기반
│   └── 스프레드시트, CSV 등
│
├── 반정형 데이터 (Semi-Structured Data)
│   ├── JSON / XML / YAML
│   ├── 로그 파일 (Log)
│   └── HTML / RSS / API 응답
│
└── 비정형 데이터 (Unstructured Data)
    ├── 텍스트 (게시글, 댓글, 리뷰)
    ├── 멀티미디어 (이미지, 영상, 음성)
    └── 소셜 활동 (감정, 관계망 등)
```

---

## 1️⃣ 정형 데이터 (Structured Data)

> **정의:** 고정된 스키마(Schema)와 행·열 구조를 가진 데이터.  
> 관계형 데이터베이스(RDBMS)에 저장되며, SQL로 즉시 처리 가능.

### 핵심 특성

| 특성 | 내용 |
|------|------|
| 구조 | 행(Row) × 열(Column) 2차원 테이블 |
| 스키마 | 사전 정의 필수 (데이터 타입, 길이, 제약조건) |
| 저장소 | Oracle, MySQL, PostgreSQL, MS-SQL |
| 처리 언어 | SQL (SELECT, JOIN, GROUP BY 등) |
| 데이터 밀도 | 높음 (결측값 최소화) |

### 📌 예시 1 — 주식 거래 데이터 (KOSPI/NASDAQ)

**구조 설명:** 매 거래마다 종목코드, 시간, 가격, 거래량이 고정 컬럼에 저장됨

| 거래ID | 종목코드 | 종목명 | 거래일시 | 시가 | 고가 | 저가 | 종가 | 거래량 | 등락률(%) |
|--------|----------|--------|----------|------|------|------|------|--------|----------|
| TXN001 | 005930 | 삼성전자 | 2024-03-15 09:00:01 | 71,200 | 72,100 | 70,800 | 71,900 | 12,345,000 | +0.98 |
| TXN002 | 035720 | 카카오 | 2024-03-15 09:00:03 | 48,300 | 48,500 | 47,900 | 48,100 | 3,210,000 | -0.41 |
| TXN003 | 000660 | SK하이닉스 | 2024-03-15 09:00:05 | 138,500 | 140,200 | 138,000 | 139,800 | 2,876,000 | +0.94 |
| TXN004 | AAPL | Apple Inc. | 2024-03-15 09:30:00 | 171.20 | 173.50 | 170.80 | 172.90 | 45,231,000 | +0.99 |
| TXN005 | TSLA | Tesla Inc. | 2024-03-15 09:30:02 | 178.50 | 180.20 | 177.10 | 179.80 | 28,450,000 | +0.73 |

**DB 스키마 정의:**
```sql
CREATE TABLE stock_transaction (
    trade_id     VARCHAR(10)    PRIMARY KEY,
    ticker       VARCHAR(10)    NOT NULL,
    stock_name   VARCHAR(50)    NOT NULL,
    trade_time   DATETIME       NOT NULL,
    open_price   DECIMAL(12,2)  NOT NULL,
    high_price   DECIMAL(12,2)  NOT NULL,
    low_price    DECIMAL(12,2)  NOT NULL,
    close_price  DECIMAL(12,2)  NOT NULL,
    volume       BIGINT         NOT NULL,
    change_rate  DECIMAL(5,2)   DEFAULT 0.00
);
```

## 2️⃣ 반정형 데이터 (Semi-Structured Data)

> **정의:** 완전한 스키마는 없지만, 태그·키·구분자 등의 **자기기술적(Self-Describing) 구조**를 포함하는 데이터.  
> 구조가 유연하며 필드가 가변적.

### 핵심 특성

| 특성 | 내용 |
|------|------|
| 구조 | 계층형 (트리), 키-값 쌍, 태그 기반 |
| 스키마 | 없거나 느슨함 (Schemaless / Schema-on-Read) |
| 저장소 | MongoDB, Elasticsearch, HBase, S3 |
| 처리 언어 | JSONPath, XPath, jq, NoSQL 쿼리 |
| 특징 | 필드 추가·삭제 자유, 중첩 구조 가능 |

## 3️⃣ 비정형 데이터 (Unstructured Data)

> **정의:** 고정된 구조나 스키마가 없어, 사전 정의된 데이터 모델로 저장하기 어려운 데이터.  
> 전체 데이터의 약 **80~90%** 를 차지하며, AI/NLP 기술로 분석.

### 핵심 특성

| 특성 | 내용 |
|------|------|
| 구조 | 없음 (자유 형식) |
| 스키마 | 없음 |
| 저장소 | HDFS, S3, Blob Storage, NoSQL |
| 처리 기술 | NLP, CV, STT, 딥러닝, 감성분석 |
| 데이터 밀도 | 낮음 (희소성 높음) |

| CCTV 영상 | 현관 카메라 | MP4 (H.264) | 1.2GB/시간 | 실시간 (24fps) | 객체탐지 (YOLO) | 침입자, 방문자 수 |
| 음성 명령 | 스마트스피커 | WAV | 50~200KB/건 | 이벤트 발생 시 | STT + NLU | 명령어, 의도 분류 |

| 초음파 이미지 | 의료 IoT | DICOM | 20~50MB/장 | 검사 시 | CNN (ResNet) | 병변 위치, 크기 |

### 📌 예시 11 — 페이스북·싸이월드 소셜 관계 데이터 (그래프)

## 🔄 데이터 유형 종합 비교표

| 구분 | 정형 데이터 | 반정형 데이터 | 비정형 데이터 |

| **처리 도구** | SQL | JSONPath, XPath, jq | NLP, CV, 딥러닝 |
| **전체 데이터 비중** | ~10% | ~10% | **~80%** |
| **싸이월드 예시** | 회원정보, 결제내역 | 방문 로그, 설정 XML | 다이어리, 사진, 음악 |

| **IoT 예시** | 수치 센서값 (DB) | MQTT JSON 페이로드 | CCTV 영상, 음성, 파형 |
| **주식 예시** | 거래 체결 내역 | 시세 API 응답 XML | 애널리스트 리포트 (PDF) |
| **분석 난이도** | ⭐ 낮음 | ⭐⭐ 중간 | ⭐⭐⭐ 높음 |
| **정보 밀도** | 높음 | 중간 | 낮음 |

## 💡 핵심 정리 (시험 포인트)

| 포인트 | 내용 |
|--------|------|
| 정형의 핵심 | **스키마 고정** + SQL 처리 가능 |
