# 🧬 유전자 알고리즘(GA)을 활용한 최대 시청률 최적화

---

## 1. 개요 및 문제 정의

### 왜 유전자 알고리즘인가?

시청률 최대화는 수많은 변수(방영 시간, 장르, 출연진, 광고 횟수, 경쟁 채널 편성 등)가 복합적으로 얽혀있는 **다변수 비선형 최적화 문제**입니다. 기존의 수리적 최적화(선형 계획법 등)로는 해결이 어려우며, 유전자 알고리즘은 이러한 복잡한 탐색 공간에서 **전역 최적해(Global Optimum)** 를 효과적으로 탐색할 수 있습니다.

### 목적 함수 (Fitness Function)

```
최대화: F(x) = 시청률(%)
```

**결정 변수 (유전자 = Gene):**

| 유전자 번호 | 변수명 | 범위 | 설명 |
|:-----------:|--------|------|------|
| G1 | 방영 시간대 | 1~24 (시) | 프로그램 시작 시간 |
| G2 | 장르 코드 | 1~5 | 1:드라마, 2:예능, 3:뉴스, 4:스포츠, 5:교양 |
| G3 | 주연 배우 등급 | 1~10 | 배우 인지도 점수 |
| G4 | 에피소드 길이 | 30~120 (분) | 방영 길이 |
| G5 | 광고 삽입 횟수 | 0~10 (회) | 중간 광고 횟수 |

**염색체 = Chromosome = [G1, G2, G3, G4, G5]**

---

## 2. 시청률 추정 함수 (Fitness Function 설계)

시청률을 추정하는 함수를 다음과 같이 정의합니다.

```
Rating(%) = base_score
          + time_bonus(G1)
          + genre_score(G2)
          + (G3 × 0.5)
          - length_penalty(G4)
          - (G5 × 0.3)
```

### 시간대별 보너스 `time_bonus(G1)`

| 시간대 | 시간 범위 | 보너스 |
|--------|-----------|--------|
| 골든타임 | 20~22시 | +5.0 |
| 프라임타임 | 22~24시 | +3.5 |
| 오후 | 14~18시 | +1.5 |
| 심야 | 00~06시 | -2.0 |
| 기타 | 나머지 | 0.0 |

### 장르별 점수 `genre_score(G2)`

| 코드 | 장르 | 점수 |
|------|------|------|
| 1 | 드라마 | +4.0 |
| 2 | 예능 | +3.5 |
| 3 | 뉴스 | +2.0 |
| 4 | 스포츠 | +1.5 |
| 5 | 교양 | +1.0 |

### 길이 패널티 `length_penalty(G4)`

```
length_penalty = 0 if 45 ≤ G4 ≤ 75  (최적 구간)
               = (G4 - 75) × 0.05 if G4 > 75
               = (45 - G4) × 0.04 if G4 < 45
```

**base_score = 3.0** (기본 시청률)

## 3. 유전자 알고리즘 전체 흐름

```
[초기 집단 생성]
        ↓
[적합도 평가 - Fitness 계산]
        ↓
[선택 - Selection (룰렛휠)]
        ↓
[교차 - Crossover (단일점 교차)]
        ↓
[돌연변이 - Mutation]
        ↓
[새로운 세대 생성]
        ↓
[종료 조건 만족?] ──YES──→ [최적해 출력]
        │ NO
        └──────────────────────↑
```

## 4. 수치 예시: 세대별 진화 과정

### Step 1: 초기 집단 생성 (세대 0, 집단 크기 = 6)

염색체 = [G1(시간대), G2(장르), G3(배우등급), G4(길이), G5(광고)]

| 개체 번호 | G1 | G2 | G3 | G4 | G5 | Fitness (시청률%) |
|:---------:|:--:|:--:|:--:|:--:|:--:|:-----------------:|
| C1 | **21** | **1** | 7 | 60 | 3 | 3.0+5.0+4.0+3.5+0.0−0.9 = **14.6** |
| C2 | 10 | 2 | 5 | 90 | 5 | 3.0+0.0+3.5+2.5−0.75−1.5 = **6.75** |
| C3 | 22 | 1 | 9 | 70 | 2 | 3.0+3.5+4.0+4.5+0.0−0.6 = **14.4** |
| C4 | 03 | 5 | 3 | 120 | 8 | 3.0−2.0+1.0+1.5−2.25−2.4 = **-1.15** |
| C5 | 21 | 2 | 8 | 55 | 1 | 3.0+5.0+3.5+4.0+0.0−0.3 = **15.2** |
| C6 | 15 | 3 | 4 | 45 | 4 | 3.0+1.5+2.0+2.0+0.0−1.2 = **7.3** |

> **계산 예시 (C5):**
> - time_bonus(21) = +5.0 (골든타임)
> - genre_score(2) = +3.5 (예능)
> - actor = 8 × 0.5 = +4.0
> - length_penalty(55) = 0 (최적 구간 45~75)
> - ad_penalty = 1 × 0.3 = −0.3
> - **Total = 3.0 + 5.0 + 3.5 + 4.0 + 0.0 − 0.3 = 15.2%**

---

### Step 2: 선택 (룰렛휠 Selection)

음수 적합도 제거 후 양수만 사용. 총 Fitness 합 = 14.6+6.75+14.4+15.2+7.3 = **58.25**

| 개체 | Fitness | 선택 확률 | 누적 확률 |
|------|---------|-----------|-----------|
| C1 | 14.6 | 25.1% | 25.1% |
| C2 | 6.75 | 11.6% | 36.7% |
| C3 | 14.4 | 24.7% | 61.4% |
| C5 | 15.2 | **26.1%** | 87.5% |
| C6 | 7.3 | 12.5% | 100.0% |

룰렛 스핀 (난수 발생) → **C5, C1, C5, C3** 선택 (높은 적합도 개체가 더 자주 선택됨)

---

### Step 3: 교차 (Single-Point Crossover)

**교차율 = 0.8**, 교차점 = 유전자 3번 이후

**부모쌍 1: C5 × C1**

```
C5 = [21 | 2 | 8 | 55 | 1]
C1 = [21 | 1 | 7 | 60 | 3]
교차점: ↑↑↑↑ 3번째 이후 교환

자식 D1 = [21, 2, 8, | 60, 3]  → Fitness 계산
자식 D2 = [21, 1, 7, | 55, 1]  → Fitness 계산
```

**D1 Fitness:**
- time_bonus(21)=+5.0, genre(2)=+3.5, actor=8×0.5=+4.0
- length_penalty(60)=0, ad=3×0.3=−0.9
- **D1 = 3.0+5.0+3.5+4.0+0.0−0.9 = 14.6%**

**D2 Fitness:**
- time_bonus(21)=+5.0, genre(1)=+4.0, actor=7×0.5=+3.5
- length_penalty(55)=0, ad=1×0.3=−0.3
- **D2 = 3.0+5.0+4.0+3.5+0.0−0.3 = 15.2%**

**부모쌍 2: C5 × C3**

```
C5 = [21 | 2 | 8 | 55 | 1]
C3 = [22 | 1 | 9 | 70 | 2]
교차점: 2번째 이후 교환

자식 D3 = [21, 2, | 9, 70, 2]
자식 D4 = [22, 1, | 8, 55, 1]
```

**D3 Fitness:**
- time_bonus(21)=+5.0, genre(2)=+3.5, actor=9×0.5=+4.5
- length_penalty(70)=0, ad=2×0.3=−0.6
- **D3 = 3.0+5.0+3.5+4.5+0.0−0.6 = 15.4%** ⭐ 신기록!

**D4 Fitness:**
- time_bonus(22)=+3.5, genre(1)=+4.0, actor=8×0.5=+4.0
- length_penalty(55)=0, ad=1×0.3=−0.3
- **D4 = 3.0+3.5+4.0+4.0+0.0−0.3 = 14.2%**

---

### Step 4: 돌연변이 (Mutation)

**돌연변이율 = 0.05 (5%)**

새로운 집단 [D1, D2, D3, D4] 중 랜덤으로 유전자 1개 변이 발생.

**D2에 돌연변이 적용 (G5: 광고 1 → 0으로 변이)**

```
D2 원본  = [21, 1, 7, 55, 1]  → Fitness = 15.2%
D2 돌연변이 = [21, 1, 7, 55, 0]  → Fitness = ?
```

- ad=0×0.3=0.0
- **D2' = 3.0+5.0+4.0+3.5+0.0−0.0 = 15.5%** ⭐⭐ 최고!

---

### Step 5: 세대 1 최종 집단

| 개체 | 염색체 [G1,G2,G3,G4,G5] | Fitness |
|------|--------------------------|---------|
| D1 | [21, 2, 8, 60, 3] | 14.6% |
| **D2'** | **[21, 1, 7, 55, 0]** | **15.5%** |
| D3 | [21, 2, 9, 70, 2] | 15.4% |
| D4 | [22, 1, 8, 55, 1] | 14.2% |

> 세대 0 최고: **15.2%** → 세대 1 최고: **15.5%** (개선됨 ✅)

## 6. 최적해 해석

```
최적 염색체 = [21, 1, 10, 55, 0]
```

| 유전자 | 값 | 해석 |
|--------|----|----|
| G1 = 21 | 21시 방영 | 골든타임 (20~22시) 시작 |
| G2 = 1 | 드라마 | 가장 높은 장르 점수 |
| G3 = 10 | 최고 등급 배우 | 배우 인지도 만점 |
| G4 = 55 | 55분 | 최적 길이 구간 (45~75분) |
| G5 = 0 | 광고 없음 | 시청자 이탈 최소화 |

**최적 예상 시청률:**

```
F* = 3.0 + 5.0(골든타임) + 4.0(드라마) + 5.0(배우10×0.5) + 0(패널티없음) − 0(광고없음)
   = 17.0%
```

## 7. 파라미터 설계 정리

| 파라미터 | 설정값 | 근거 |
|----------|--------|------|
| 집단 크기 (Population Size) | 50~200 | 탐색 다양성과 속도의 균형 |
| 세대 수 (Generations) | 100~500 | 수렴까지 충분한 반복 |
| 교차율 (Crossover Rate) | 0.7~0.9 | 높을수록 탐색 넓어짐 |
| 돌연변이율 (Mutation Rate) | 0.01~0.05 | 너무 높으면 무작위 탐색과 동일 |
| 선택 방식 | 룰렛휠 / 토너먼트 | 적합도 비례 선택 |
| 엘리트 보존 (Elitism) | 상위 2~5개 보존 | 최적해 손실 방지 |

## 8. 유전자 알고리즘의 장단점 (시청률 최적화 맥락)

### ✅ 장점
- 수학적 미분 없이도 **비선형 복잡 문제** 탐색 가능
- 여러 변수가 동시에 상호작용하는 경우에도 유효
- 전역 최적해(Global Optimum) 탐색 능력 우수
- 새로운 변수(SNS 반응, 날씨 등) 추가 확장 용이

### ⚠️ 단점
- **수렴 보장 없음** (확률적 탐색)
- 파라미터(교차율, 돌연변이율 등) 설정이 결과에 민감
- 계산 비용이 높을 수 있음 (집단 크기 × 세대 수)
- 지역 최적해(Local Optimum)에 빠질 위험 존재

## 9. 실무 확장 방안

```
실제 적용 시 추가 고려 변수:
  ├── 경쟁 채널 편성표 (타 채널 시청률 영향)
  ├── 계절/요일 효과 (주말 vs 평일)
  ├── SNS 화제성 지수 (사전 마케팅)
  ├── 날씨 데이터 (야외 활동 대체재 효과)
  └── 과거 시청률 시계열 데이터 (ARIMA 결합)
```

> 유전자 알고리즘은 이러한 다변수 환경에서도  
> **하이퍼파라미터 탐색(HPO)** 방식으로 확장 가능하며,  
> 딥러닝 시청률 예측 모델과 결합하면 더욱 정교한 최적화가 가능합니다.
